01 — Structure HTML
HTML décrit le sens et la structure du contenu. Le navigateur construit un arbre DOM à partir de ce balisage — c'est ce que JS et CSS manipulent. Depuis HTML5, on privilégie les balises sémantiques (qui disent ce qu'est le contenu) plutôt que des <div> partout, pour l'accessibilité et le SEO.
Squelette minimal
<!DOCTYPE html>
<html lang="fr">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport"
          content="width=device-width, initial-scale=1">
    <title>Titre de la page</title>
    <link rel="stylesheet" href="style.css">
  </head>
  <body>
    <!-- contenu ici -->
    <script src="app.js"></script> <!-- en bas -->
  </body>
</html>

Le <script> en bas de <body> garantit que le DOM est prêt avant l'exécution. Sinon, utiliser l'attribut defer.

Balises sémantiques HTML5

Ces balises remplacent les <div class="header"> : elles communiquent leur rôle aux navigateurs et aux lecteurs d'écran.

<header> <nav> <main> <footer> <section> <article> <aside> <figure> <figcaption> <blockquote> <details> <summary> <dialog> <time> <mark> <strong> <em>
Règle : <h1> unique par page, <main> unique, hiérarchie h1→h2→h3 sans sauter de niveau.
Liens & médias
<!-- Lien interne / ancre / externe -->
<a href="page.html">Interne</a>
<a href="#section-id">Ancre</a>
<a href="https://…" target="_blank"
   rel="noopener noreferrer">Ext.</a>

<!-- Image (avec dimensions = évite layout shift) -->
<img src="photo.webp" alt="Description précise"
     width="800" height="450" loading="lazy">

<!-- Vidéo -->
<video src="v.mp4" controls muted loop></video>
Formulaires

Toujours lier <label> et <input> via for/id — obligatoire pour l'accessibilité.

<form action="/" method="post">
  <label for="email">Email</label>
  <input type="email" id="email"
         name="email" required>
  <select name="choix">
    <option value="a">Option A</option>
  </select>
  <textarea rows="4" name="msg"></textarea>
  <button type="submit">Envoyer</button>
</form>

02 — CSS Fondamentaux

Box model

Chaque élément est une boîte : contenu → padding → border → margin. Avec border-box, la width inclut padding et border — comportement attendu dans 99% des cas. 

box-sizing: border-box; /* toujours */
width: 300px;
height: auto;          /* s'adapte au contenu */
padding: 16px 24px;   /* vertical horizontal */
border: 1px solid #ccc;
margin: 0 auto;       /* centrage horizontal */
overflow: hidden | scroll | auto;
min-width: 0;         /* ⚠ fix overflow flex */
max-width: 1200px;
aspect-ratio: 16 / 9;
Display & position
display: blockprend toute la largeur
display: inlinecoule dans le texte
display: inline-blockinline + dimensions
display: flexflexbox (1D)
display: gridgrille (2D)
display: noneinvisible + absent du flux
POSITION
staticdéfaut, suit le flux
relativedécalé par rapport à soi
absoluteparent relative le plus proche
fixedrelatif à la fenêtre
stickycolle au scroll jusqu'à threshold

Pour absolute/fixed : utiliser top right bottom left ou le raccourci inset: 0. Le z-index ne fonctionne qu'avec position ≠ static.

03 — Flexbox (1D)
Flexbox est idéal pour aligner des éléments sur un axe (ligne ou colonne). L'axe principal (main axis) est défini par flex-direction. L'axe secondaire (cross axis) lui est perpendiculaire. justify-content agit sur le main axis, align-items sur le cross axis.
Conteneur (parent)
display: flex;

/* direction de l'axe principal */
flex-direction: row | column | row-reverse | column-reverse;
flex-wrap: nowrap | wrap | wrap-reverse;
/* shorthand des deux : */
flex-flow: row wrap;

/* répartition sur l'axe principal */
justify-content: flex-start | flex-end | center
  | space-between | space-around | space-evenly;

/* alignement sur l'axe secondaire */
align-items: stretch | center | flex-start | flex-end | baseline;

/* si wrap : alignement des lignes */
align-content: flex-start | center | space-between…;

gap: 16px;         /* espacement entre enfants */
gap: 16px 24px;   /* row-gap column-gap */
Enfants (items)
/* flex: grow shrink basis */
flex: 1;          /* = 1 1 0 : grandit, rétrécit */
flex: 0 0 200px;  /* taille fixe, ne bouge pas */
flex-grow: 1;    /* absorbe l'espace libre */
flex-shrink: 0; /* refuse de rapetisser */
flex-basis: auto | 200px | 30%;

/* override de align-items pour cet enfant */
align-self: center | flex-start | stretch…;

order: 0;         /* défaut 0, peut être négatif */
min-width: 0;     /* ⚠ fix overflow texte long */
Centrage parfait :
display: flex; justify-content: center; align-items: center;
04 — CSS Grid (2D)
Grid est fait pour les mises en page à deux dimensions (lignes + colonnes). L'unité fr (fraction) représente une part de l'espace disponible. auto-fit + minmax() donne des grilles responsive sans media queries.
Conteneur grid
display: grid;

/* colonnes */
grid-template-columns: 1fr 2fr 1fr;
grid-template-columns: repeat(3, 1fr);
grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
grid-template-columns: 200px 1fr auto;

/* lignes */
grid-template-rows: auto 1fr auto;

/* zones nommées */
grid-template-areas:
  "header header header"
  "sidebar main  main  "
  "footer footer footer";

gap: 24px;
align-items: start | center | stretch;
justify-items: start | center | stretch;
place-items: center; /* shorthand */
Placement d'éléments
/* par numéros de lignes */
grid-column: 1 / 3;    /* de la ligne 1 à 3 */
grid-column: 1 / -1;   /* toute la largeur */
grid-column: span 2;   /* occupe 2 colonnes */
grid-row: 2 / 4;
grid-row: span 3;

/* par zone nommée */
grid-area: header;
grid-area: sidebar;

/* alignement de cet élément dans sa cellule */
justify-self: start | center | end | stretch;
align-self: start | center | end | stretch;
place-self: center;

Les formats modernes oklch() et color-mix() offrent une perception uniforme des couleurs. hsl() reste très lisible pour les ajustements manuels.

06 — Responsive & Variables CSS
Approche mobile-first : écrire le CSS de base pour mobile, puis ajouter des min-width breakpoints pour les écrans plus larges. Les fonctions clamp(), min(), max() permettent des mises en page fluides sans media queries.
Media queries
/* mobile-first : styles de base pour mobile */
.container { padding: 1rem; }

@media (min-width: 640px)  { /* ≥ sm  */ }
@media (min-width: 768px)  { /* ≥ md  */ }
@media (min-width: 1024px) { /* ≥ lg  */ }
@media (min-width: 1280px) { /* ≥ xl  */ }

/* préférences système */
@media (prefers-color-scheme: dark) { }
@media (prefers-reduced-motion: reduce) {
  * { animation-duration: 0.01ms !important; }
}

/* unités fluides — sans media query */
font-size: clamp(1rem, 2.5vw, 1.5rem);
width: min(90%, 1200px);
padding: max(1rem, 5vw);
Variables CSS (custom properties)

Les variables CSS sont dynamiques : elles se recalculent en temps réel et peuvent être modifiées par JS ou héritées dans le DOM. Contrairement aux variables Sass, elles fonctionnent nativement dans le navigateur. 

/* déclaration sur :root = globale */
:root {
  --color-primary: #5B4EF5;
  --color-text: hsl(220 10% 15%);
  --spacing-sm: 0.5rem;
  --spacing-md: 1rem;
  --radius: 8px;
  --shadow: 0 2px 12px rgba(0 0 0 / .1);
}

.btn {
  background: var(--color-primary);
  padding: var(--spacing-sm) var(--spacing-md);
  border-radius: var(--radius);
}

/* override local (dark mode) */
.dark { --color-primary: #8B7FF9; }

/* avec valeur de fallback */
color: var(--color-brand, #000);
07 — Transitions & Animations
Préférer animer transform et opacity : ils ne déclenchent pas de recalcul de layout (reflow), contrairement à width, height, top, margin… Les animations sur ces propriétés sont accélérées par le GPU.
Transitions
/* syntaxe complète */
transition: property duration timing-function delay;

transition: all 0.2s ease;
transition: opacity 0.3s ease,
             transform 0.3s cubic-bezier(.4, 0, .2, 1);
transition: color 0.15s, background 0.15s;

/* timings */
ease | linear | ease-in | ease-out | ease-in-out
cubic-bezier(x1, y1, x2, y2)
steps(4, end)

/* transform : toujours préféré */
transform: translateX(10px);
transform: translateY(-50%) scale(1.05);
transform: rotate(45deg) skewX(10deg);
transform-origin: center | top left | 50% 50%;
will-change: transform; /* hint GPU, à ne pas abuser */
Keyframes & animations
@keyframes fadeInUp {
  from {
    opacity: 0;
    transform: translateY(16px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.card {
  animation: fadeInUp 0.4s ease forwards;
  /* name duration timing fill-mode */
}

animation-delay: 0.1s;
animation-iteration-count: infinite | 3;
animation-direction: normal | alternate | reverse;
animation-fill-mode: forwards | backwards | both;
animation-play-state: paused | running;
